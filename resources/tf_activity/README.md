# CAP-TF-002 production resources

These versioned human resources are generated by
`scripts/prepare_tf_resources.py` and validated through the production
`normalize_signed_resource` path.

- DoRothEA is exported from Bioconductor package `dorothea` 1.18.0 and limited
  to confidence A/B/C.
- CollecTRI is retrieved from the official OmniPath interactions API and
  normalized with `decoupleR::get_collectri(organism="human",
  split_complexes=FALSE)` semantics.

Accepted files are comma-delimited for `.csv` and tab-delimited for `.tsv` or
`.txt`. Required logical columns are `source`, `target`, and `weight` (`tf`,
`target_gene`, and `gene` aliases are supported by the node). Weights must be
finite numeric nonzero values. `confidence`, `organism`, and
`resource_metadata` are optional, although production DoRothEA must provide
confidence values because the default filter retains A/B/C.

Exact duplicate rows are removed and duplicate TF-target edges with the same
weight collapse. Conflicting weights for the same normalized edge fail. A TF
model requires at least five targets overlapping the eligible signed gene
program and at least three eligible program genes.

Validate configured files with:

```bash
PYTHONPATH=src conda run -n cellstate-agent python scripts/validate_tf_resources.py
```

Copy `config/resources.env.example` to `.env.cellstate.local`, replace its
paths, and launch with `./run-cellstate-agent.sh`.
